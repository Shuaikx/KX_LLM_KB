# 投递记录录入

快速将一条新的岗位投递记录追加到 `投递记录.md`。

## 用法

`/log-job` 后可选直接附上岗位信息，或不带参数时 Claude 会询问必要字段。

## 执行步骤

1. **收集以下字段**（若用户在命令后已给出，直接解析；否则通过提问获取）：
   - **公司**（company）
   - **岗位**（position）
   - **投递渠道**（channel）
   - **工作地点 / 备注**（location / notes）
   - **投递日期**（date）：默认使用今天的日期（从 CLAUDE.md 的 `currentDate` 获取）
   - **状态**（status）：默认 `✉️`（已投递）；可选 `📝 待投递 / ✉️ 已投递 / 📞 已约面 / ⏳ 面试中 / 🎉 已拿Offer / ❌ 已拒/放弃`

2. **读取 `投递记录.md`**，找到：
   - 投递列表表格末尾，获取当前最大序号 `N`，新条目序号为 `N+1`
   - `## 详细记录` 区块末尾

3. **在投递列表表格**末尾（`---` 分隔线之前）追加一行：
   ```
   | N+1 | {公司} | {岗位} | {投递渠道} | {投递日期} | {状态} | {备注} |
   ```

4. **在详细记录区块**末尾追加新的详细条目模板：
   ```markdown
   ### N+1. {公司} · {岗位}

   - **投递日期：** {投递日期}
   - **工作地点：** {工作地点}
   - **投递渠道：** {投递渠道}
   - **岗位亮点：** （待补充）
   - **核心要求：** （待补充）
   - **匹配度分析：**
     - （待补充）

   **跟进记录：**

   | 日期 | 事件 | 备注 |
   |------|------|------|
   | {投递日期} | 投递简历 | |

   ---
   ```

5. **完成后**，输出一条确认消息，展示刚追加的表格行，提示用户可以手动补充「岗位亮点」和「匹配度分析」。
